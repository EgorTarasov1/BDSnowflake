# BigDataSnowflake
Анализ больших данных - лабораторная работа №1 - нормализация данных в снежинку

Одна из задач data engineer при работе с данными BigData трансформировать исходную модель данных источника в аналитическую модель данных. Аналитическая модель данных позволяет исследовать данные и принимать на основе полученных данных решения. Классическими универсальными схемами для анализа данных являются "звезда" и "снежинка". В лабораторной работе вам предстоит потренироваться в трансформации исходных данных из источников в модель данных снежинка.

Что необходимо сделать?

Необходимо данные источника (файлы mock_data.csv с номерами), которые представляют информацию о покупателях, продавцах, поставщиках, магазинах, товарах для домашних питомцев трансформировать в модель снежинка/звезда (факты и измерения с нормализацией).

# Структура проекта

1. **Папка sql** — содержит в себе sql-скрипты, реализующие задание лабораторной работы.
    * `import.sql`: создание исходной таблицы, соответствующей по структуре исходным .csv файлам, загрузка данных в исходную таблицу, предобработка данных.
    * `ddl.sql`: создание таблиц, соответствующих новой модели "снежинка" для распределения исходных данных.
    * `dml.sql`: заполнение таблиц модели "снежинка" данными из исходной таблицы.
    * `test.sql`: 10 тестов, которые доказывают целостность данных в новой модели.
2. **diagram.png** — скриншот диаграммы dbeaver итоговой модели "снежинка".
3. **docker-compose.yml** — docker-файл для удобного и быстрого запуска проекта.
4. **REPORT.md** — краткий отчёт по выполнению лабораторной работы.

# Быстрый запуск

1) Клонируете к себе репозиторий:

```bash
git clone https://github.com/EgorTarasov1/BDSnowflake.git
```

2) Переходите в папку проекта:

```bash
cd BDSnowflake
```

3) Поднимаете базу данных при помощи docker-контейнера:

```bash
docker compose up -d
```
